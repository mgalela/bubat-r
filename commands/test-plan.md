# bubat-r test-plan

Generate test plan sebagai companion ke plan atau plan-detail phase.

## Intent

```text
bubat-r test-plan <plan-id>
bubat-r test-plan <plan-id> --phase A
```

- Tanpa `--phase`: generate test plan untuk seluruh plan, output ke `overlays/test-plans/TP-<plan-id>.md`
- Dengan `--phase A`: generate test plan untuk satu phase, output ke `overlays/plans/<plan-id>/test-plan-phase-A.md` (companion ke phase file)

## Path Resolution

- Input: main plan file `${BUBATR_HOME}/STAGES/overlays/plans/<plan-id>.md`
- Input opsional: phase file `${BUBATR_HOME}/STAGES/overlays/plans/<plan-id>/phase-<X>.md`
- Output (full plan): `${BUBATR_HOME}/STAGES/overlays/test-plans/TP-<plan-id>.md`
- Output (per phase): `${BUBATR_HOME}/STAGES/overlays/plans/<plan-id>/test-plan-phase-<X>.md`

## Protocol

1. Baca plan file (dan phase file jika `--phase` disediakan).
2. Extract: Tasks, Acceptance criteria, Gate conditions, Risk Callouts.
3. Derive test cases dari:
   - Setiap **Gate** di task → TC (unit/integration level)
   - Setiap **Acceptance** baris → TC (phase-level acceptance)
   - Setiap **Risk Callout** → TC (regression/negative case)
4. Tulis test plan dengan struktur:

```markdown
# Test Plan: <plan-id> [Phase X]

**Companion to:** `STAGES/overlays/plans/<plan-id>.md`
**Phase:** [All | A — <title>]
**Generated by:** `bubat-r test-plan`
**Date:** YYYY-MM-DD
**Execution status:** PENDING

## Pre-conditions

[Derived dari plan header + Depends on]

## Test Cases

### TC-01: <derived dari gate/acceptance>

**Input:** ...
**Expected:** ...
**Pass criteria:**

- [ ] ...

## Deviation Log

| TC  | Status | Deviasi | Catatan |
| --- | ------ | ------- | ------- |
```

5. Update main plan `## Detail Plans` (atau phase file header): tambah link ke test plan.
6. Catat di `00-workflow-status.md` kolom baru `Test Plan` jika test plan full (bukan per-phase).

## Rule

```text
test-plan adalah executable checklist — bukan deskripsi. Setiap TC harus punya Pass criteria terukur.
```
