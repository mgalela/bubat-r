# bubat-r plan-detail

Generate per-phase detail files dari main plan. Satu file per phase, dikumpulkan dalam satu direktori.

## Intent

```text
bubat-r plan-detail <plan-id>
bubat-r plan-detail PL-ADR-20260718-001
bubat-r plan-detail STAGES/overlays/plans/PL-ADR-20260718-001.md
```

`<plan-id>` adalah filename tanpa extension, atau path ke plan file.

## Path Resolution

- Plan file: `${BUBATR_HOME}/STAGES/overlays/plans/<plan-id>.md`
- Output dir: `${BUBATR_HOME}/STAGES/overlays/plans/<plan-id>/`
  - Satu file per phase: `phase-A.md`, `phase-B.md`, dst.
  - Nama phase diambil dari heading `## Phase X — <title>` di main plan.

## Protocol

1. Baca main plan file.
2. Parse semua `## Phase X` sections.
3. Buat direktori `${BUBATR_HOME}/STAGES/overlays/plans/<plan-id>/` jika belum ada.
4. Untuk setiap phase: tulis file `phase-<letter>.md` — salin seluruh konten phase (Tasks, Acceptance, Planned changes, Executed changes, Results, Findings, Notes).
5. Header tiap phase-file:
   ```markdown
   **Plan:** `STAGES/overlays/plans/<plan-id>.md`
   **Phase:** A — <title>
   **Generated by:** `bubat-r plan-detail`
   ```
6. Update main plan `## Detail Plans` section:
   ```markdown
   ## Detail Plans

   **Directory:** `STAGES/overlays/plans/<plan-id>/`
   | Phase | File | Status |
   |---|---|---|
   | A — <title> | `phase-A.md` | ⏳ pending |
   | B — <title> | `phase-B.md` | ⏳ pending |
   ```
7. hapus konten yang sudah disalin ke phase files dari main plan file.

## Rule

Phase file bersifat executable — developer kerja dari phase file, bukan dari main plan.
Main plan tetap sebagai index dan status tracker.
